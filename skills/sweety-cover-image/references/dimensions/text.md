# 文字维度

控制封面图上的文字密度和信息层次。

## 取值

| 值 | 标题 | 副标题 | 标签 | 视觉面积 |
|-----|:----:|:------:|:----:|:--------:|
| `none` | - | - | - | 100% |
| `title-only` | ✓ | - | - | 85% |
| `title-subtitle` | ✓ | ✓ | - | 75% |
| `text-rich` | ✓ | ✓ | ✓ (2-4) | 60% |

## 详情

### none

纯视觉封面，无文字元素。

**适用场景**：
- 摄影风格封面
- 抽象艺术作品
- 纯视觉社交分享
- 标题在外部添加时

**构图**：
- 全部视觉面积可用
- 不预留文字区域
- 强调视觉隐喻

### title-only

单标题，最大冲击力。

**适用场景**：
- 大多数文章封面（默认）
- 清晰的单一信息
- 强品牌辨识度

**构图**：
- 标题：突出放置
- 预留区域：顶部或底部 15%
- 视觉辅助标题信息

**标题指南**：
- 使用来源内容或用户提供的原始标题
- 禁止自行创造或修改标题
- 匹配内容语言

### title-subtitle

标题 + 辅助上下文。

**适用场景**：
- 需要说明的技术文章
- 带集数/部分信息的系列
- 双信息内容

**构图**：
- 标题：主要元素
- 副标题：次要元素
- 预留区域：25%
- 标题/副标题间清晰层次

**标题指南**：
- 使用来源内容或用户提供的原始标题
- 禁止自行创造或修改标题

**副标题指南**：
- 澄清或提供标题上下文
- 可包含系列名、作者、日期
- 比标题更小、更次要

### text-rich

信息密集的封面，包含多种文字元素。

**适用场景**：
- 信息图风格封面
- 包含详细信息的活动公告
- 包含功能特性的推广材料
- 包含多个要点的内容

**构图**：
- 标题：主要焦点
- 副标题：辅助信息
- 标签：2-4 个关键词标签
- 预留区域：40%
- 清晰的视觉层次

**标题指南**：
- 使用来源内容或用户提供的原始标题
- 禁止自行创造或修改标题

**标签指南**：
- 最多 2-4 个标签
- 简短关键词（每个 1-2 词）
- 以徽章/标签形式放置
- 可突出：类别、日期、作者、关键特性

## 类型兼容性

| 类型 | none | title-only | title-subtitle | text-rich |
|------|:----:|:----------:|:--------------:|:---------:|
| hero | ✓ | ✓✓ | ✓✓ | ✓ |
| conceptual | ✓✓ | ✓✓ | ✓ | ✓ |
| typography | ✗ | ✓ | ✓✓ | ✓✓ |
| metaphor | ✓✓ | ✓ | ✓ | ✗ |
| scene | ✓✓ | ✓ | ✓ | ✗ |
| minimal | ✓✓ | ✓✓ | ✓ | ✗ |

✓✓ = 强烈推荐 | ✓ = 兼容 | ✗ = 不推荐

## 自动选择

未指定 `--text` 时，根据内容信号选择：

| 内容信号 | 文字级别 |
|----------|----------|
| 纯视觉、摄影、抽象、艺术 | `none` |
| 文章、博客、标准封面 | `title-only` |
| 系列、教程、技术文章（需上下文） | `title-subtitle` |
| 公告、功能介绍、多要点、信息图 | `text-rich` |

默认：`title-only`
