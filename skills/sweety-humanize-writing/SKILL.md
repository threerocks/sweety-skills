# 去AI味写作

这个 skill 用于把中文文本从“完整但没个性”的 AI 腔，改成更像真实作者写出来的文字。核心目标是保留事实、压缩废话、增加具体判断、恢复语体节奏，而不是骗检测器。

test -f .sweety-skills/sweety-humanize-writing/EXTEND.md && echo "project"
test -f "${XDG_CONFIG_HOME:-$HOME/.config}/sweety-skills/sweety-humanize-writing/EXTEND.md" && echo "xdg"

## 工作流

### 1. 明确任务边界

先判断用户要的是哪一种：

| 用户意图 | 输出 |
|----------|------|
| “去 AI 味”“润色得自然点” | 直接输出改写后全文 |
| “诊断哪里像 AI” | 输出问题清单和改写建议 |
| “对比修改” | 输出改写版，再列 3-6 条关键变化 |
| “保留结构/不要改事实/不要口语化” | 只做句子级和词语级清洁 |
| “学习我的文风”“按我以前的文章写” | 先提取风格锚点，再改写或生成 |

如果原文、目标平台、读者或语气完全不清楚，先问一个最关键的问题。若用户只给原文，默认保留原意和原结构，做中等力度改写。

### 2. 识别语体

改写前先判定文本场景，不要把所有文本都改成朋友聊天：

| 场景 | 改写力度 |
|------|----------|
| 社媒短文、Threads/X、小红书、朋友圈 | 可更口语、更短句、更有主观判断 |
| 公众号、知识类长文、口播稿 | 保留信息密度，增强段落节奏和场景感 |
| 动漫剧情解析、短视频文案 | 事实和事件顺序优先，短句可以有，但不能为了节奏牺牲通顺 |
| 技术博客、产品说明、商业文章 | 保留专业表达，删除套话和抽象拔高 |
| 学术、公文、法律、政策文本 | 保守处理，不强行口语化 |

### 3. 诊断 AI 味

优先检查这些信号：

- 模板开头：在当今社会、随着科技发展、众所周知、本文将从。
- 机械连接：首先、其次、最后、此外、然而、综上所述、值得注意的是。
- 重复解释：一句话讲观点，下一句只是换个说法解释同一件事。
- 抽象空词：高效、赋能、革命性、显著提升、深度融合、复杂生态。
- 过度完整：每段都有承上启下，每个结尾都拔高，每个观点都不敢下判断。
- 均匀句长：每句都差不多长，读起来像模板排队。
- 符号脚手架：滥用冒号、破折号、括号、分号，或把句号打得过密，像在输出提纲。
- 客服腔：希望本文对您有所帮助、期待您的反馈、感谢阅读。
- 伪人味：硬塞“你懂的”“说白了”“老实说”，但没有真实判断或细节。
- 语言指纹：高频“最/真正/本质上”、否定式煽情、虚假范围、过度拔高、没有证据链的论证。
- 过度规整：连接词密度异常高、标点过于统一、句式结构太完美、段落排版像模板。
- 风格稀释：用户明明提供了个人样本，输出却回到通用公文腔、营销腔或知乎腔。

详细规则见 [references/rules.md](references/rules.md)。示例见 [references/examples.md](references/examples.md)。

### 4. 可选风格校准

当用户提供本人旧文、账号样本或明确要求“像我写的”时，先做风格校准，不要直接改写：

- 优先使用 3-5 篇能代表目标风格的样本；总字数不足时也可以做弱校准，但要降低确定性。
- 从样本里提取风格锚点：高频过渡词、标志性句式、惯用比喻、句长分布、段落收束方式、常见禁用项。
- 把主观反馈翻译成可执行约束。例如“这里不像我”要落成“主语前置”“少用副词”“结尾不要拔高”。
- 只学习表达方式，不复制样本事实、经历、隐私或来源不明的判断。

### 5. 三遍改写

第一遍做减法：

- 删除问候、铺垫、总结升华和空泛承诺。
- 合并重复句，只保留带来新信息的句子。
- 把“陈述 + 解释 + 再解释”压成一个清楚判断。

第二遍做具体化：

- 用人、动作、场景、数字、对象替代抽象形容词。
- 只使用原文已有事实；没有材料时，不编造经历、案例、数据或情绪。
- 如果缺少支撑，改成更克制的判断，而不是硬写得像亲历。
- 检查每一句是否符合中文习惯。为了避开某个句式而写出别扭句、病句或事实含混，必须重写整句。
- 检查句子逻辑门禁：两边能不能比较，情绪词有没有事实证据，读者会不会先被句式绊住。

第三遍做节奏：

- 长短句交错。连续三句长度相近时，拆一句或合一句。
- 先按语义调整句子，再加标点；不要靠冒号、破折号、分号制造层次。
- 句号只放在意思真正收住的位置。短句可以有，但不要把每个短意群都切成独立句。
- 列表不要天然写成“第一、第二、第三”，除非用户要清晰教程。
- 加入口语插入语要少量，并且必须服务于语气，不要为了“人味”而撒调料。

第四遍做核验：

- 具体数据、统计、人物引语、文献引用和独特事实，必须保留来源意识；没有来源时标注需要补证或改成克制表达。
- 检查论证链是否成立，不要让看似顺滑的句子替代事实、因果和证据。
- 不为“更像人”添加未经用户提供的亲历、案例、采访、数据或情绪。

### 6. 输出规则

默认只输出改写后的正文，不解释修改过程。

用户需要诊断时，使用这个结构：

```markdown
## 改写后
[全文]

## 关键修改
- [问题] -> [处理方式]
- [问题] -> [处理方式]
```

用户要求多版本时，最多给 3 个版本：

- 克制版：保留原结构，只清理机器腔。
- 自然版：默认推荐，调整段落和节奏。
- 犀利版：观点更锋利，适合社媒或评论文章。

## 硬性边界

- 不承诺“绕过 AI 检测”。
- 不伪造个人经历、采访、数据、引用、来源。
- 不把正式文档强行改成网络口语。
- 不为了去 AI 味而牺牲准确性、逻辑和可读性。
- 不保留来源不明的绝对化判断；必要时改成“可能”“更像是”“在多数情况下”。
- 不把本 skill 用作规避检测、伪装来源或证明“不是 AI”的工具；目标是改善表达质量和事实可靠性。
- 不故意制造错别字、半角标点混用、异常空格或排版错误来假装“人工痕迹”。
