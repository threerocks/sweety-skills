# 去AI味写作规则

## 底层原则

去 AI 味的重点：不要把文字弄得更随便；要恢复四件事：

- 具体对象：写清楚是谁、在哪、做了什么、结果如何。
- 作者判断：允许有取舍、有边界、有轻微不确定。
- 真实节奏：短句、长句、停顿、转折自然混合。
- 场景匹配：公众号、社媒、技术文、公文的“自然”不是同一种自然。
- 语言指纹：识别文本反复暴露的句法、词频、论证和格式习惯，而不是只盯几个禁用词。
- 风格来源：如果用户有自己的旧文样本，优先从样本提取风格锚点；没有样本时才使用默认规则。

## 优先级

按这个顺序判断输出质量：

1. 事实准确，不能把人物、事件、因果、时间线写错。
2. 中文通顺，不能为了避开 AI 句式写出病句或别扭句。
3. 场景合适，口播、公众号、技术文、短评要有不同语气。
4. 去掉 AI 味，包括模板词、符号脚手架、机械转折和空泛抽象。

如果第 4 点和前 3 点冲突，优先保留事实、通顺和场景。不要做词语级机械替换。例如为了避开“不是...而是...”，不能把句子硬改成“不通顺的 A，是 B”。应该重写整个表达。

## 处理顺序

1. 删模板：去掉问候、背景废话、空泛总结、强行号召。
2. 删重复：后一句如果只是解释前一句，合并或删除。
3. 换具体：把抽象形容词换成动作、数字、场景、对象。
4. 改语态：优先主动语态，但不要破坏正式语境。
5. 调标点：按停顿和语义层级重排标点，不用符号强造结构。
6. 破节奏：避免整篇等长句、整齐并列、机械列表。
7. 收口气：减少绝对化、客服腔和过度取悦。

## 常见 AI 味与处理

| AI 味 | 更自然的处理方向 |
|-------|------------------|
| 在当今社会 | 直接写事件、问题或观察 |
| 随着科技的发展 | 写具体技术、时间点或变化 |
| 首先/其次/最后 | 换成自然段推进；必要时用“先说最重要的” |
| 此外 | 还有一点、对了、再补一句，或直接另起段 |
| 然而 | 但问题是、可实际情况是、不过 |
| 因此 | 所以、这就导致、结果是 |
| 综上所述 | 说白了、到这里可以看出，或直接删除 |
| 值得注意的是 | 直接说重点，或改成更具体的观察 |
| 这是一个...的过程 | 改成具体动作链条 |
| 不仅 X，也 Y | 只有确实形成递进时才保留 |
| 从 X 到 Y | 确认 X/Y 真有跨度，否则拆开写 |
| 问题是：X | 改成自然问句或普通陈述 |
| 结论是：X | 直接写判断，或让判断自然落在段末 |
| 原因有三：X | 真要分项时另起段；普通开头不要这样写 |
| 不是 X，而是 Y | 尽量改成直接判断，少用二分包装 |
| 真正/其实/本质上 | 少量使用。能用具体场景和因果说明时，不靠加强词撑判断 |
| 最狠的是/最核心的是 | 能具体说看点、剧情走向、人物选择时，不靠“最”硬拔高度 |
| X 败了，Y 散了 | 只有语义准确且读者能立刻理解时使用；否则改成明确事件 |

## 语言指纹排查

AI 味常常不是某个词造成的，而是一组稳定的语言指纹叠在一起。诊断时按下面顺序看。

### 过度拔高

把普通变化写成时代拐点，把局部事件写成历史时刻，是典型 AI 味。

- 高风险：历史性、关键时刻、高光、决定性、革命性、里程碑、深刻改变。
- 处理：先问这件事到底改变了谁的什么动作；如果答不上来，就降级为具体影响或观察。
- 不要把“小镇很普通”改成“小镇象征韧性”；除非原文真的有事件支撑。

### 否定式煽情

“这不仅仅是 X，而是 Y”很容易把事实包装成情绪承诺。

- 高风险：不只是一双跑鞋，而是一种生活方式；不只是工具，而是创意伙伴。
- 处理：先写可验证属性，再写有限判断。
- 如果 X 和 Y 不在同一层级，拆成两句，不要硬做反转。

### 虚假范围

“从 X 到 Y”只有在 X 和 Y 真的构成范围、跨度或演进时才成立。

- 高风险：从解决问题的工具，到科学发现的艺术表达。
- 自检：X/Y 是否同类？中间有没有真实跨度？读者能不能自然理解为什么放在一起？
- 处理：无真实跨度时，改成并列事实或删除其中一端。

### 高频强度词

“最”“真正”“本质上”“核心”如果连续出现，会让文章像在靠语气压人。

- 每段最多保留一个强度词，且后面必须有动作、数据、机制或例子支撑。
- 删除强度词后句子不成立，说明判断本身太虚。

### 论证无来源

AI 会把“看起来合理”的推理写成事实。涉及真实性时，不只改文风，还要看证据链。

- 数据和统计：没有来源就不要写成确定数字。
- 引语和人物：没有原始出处就不要写成直接引用。
- 文献和论文：题名、作者、DOI、年份要能对应得上；找不到时标注待核验。
- 独特事实：用更克制的话处理，例如“有待核实”“公开资料中暂未看到明确来源”。

## 风格样本校准

当用户要求“像我”“像这个账号”“按我的旧文语气”时，先抽取样本规则，再动笔。

### 样本要求

- 理想输入：3-5 篇目标风格文本，总字数 1500 字以上。
- 样本要覆盖不同句型：说明性段落、议论句、口语短句、修辞长句。
- 样本不足时可以弱校准，但要避免声称已经完整复刻风格。

### 风格锚点

从样本里提取这些锚点：

- 高频过渡词：作者怎么换话题、转折、补充。
- 标志性句式：主语放前还是放后，喜欢短判断还是长铺陈。
- 惯用比喻结构：是生活化比喻、技术类比，还是讽刺暗喻。
- 句长分布：连续短句多，还是长短句交错。
- 段落收束：用判断收尾、问题收尾，还是具体画面收尾。
- 禁用项：作者明显不用的词、符号、被动语态、连续顿号、营销口号。

### 反馈闭环

用户说“不像”“太 AI”“太顺”“太用力”时，不要只道歉重写。先把反馈翻译成约束：

- “不像我” -> 对照样本找句式、词频、段落收束差异。
- “太顺” -> 减少模板转折，允许轻微跳跃，但不制造病句。
- “太用力” -> 删除强度词、副词和煽情反转。
- “太像报告” -> 拆掉标题化推进，改成自然段叙述。
- “太口语” -> 恢复专业名词和完整句，但保留具体对象。

可用 A/B/C 三版对比，但每版必须有明确差异，例如“更短句”“更克制”“更像样本里的评论腔”。不要给三个只换词不换结构的版本。

## Prompt 输入清洁

当用户让你帮他写提示词或从提示词源头减少 AI 味时，优先补齐边界，不要只加“像真人”。

- 角色：行业、经验、能力边界要具体，但不要虚构身份。
- 需求：受众、目标、篇幅、平台、禁用项要明确。
- 示例：提供 2-3 个同领域、同风格、质量可对照的样例。
- 修正：指出要改哪一段、保留哪一部分、压缩到什么程度。
- 输出：格式、标题层级、是否保留列表、是否需要引用要提前说清。
- 附加约束：把“不用专业术语”改成“专业术语第一次出现时用一句话解释”；把“更自然”改成“少用四字词，保留具体动作，结尾不升华”。

弱提示词：

> 帮我写得自然一点。

更可执行：

> 保留原文事实和段落顺序，删掉空泛总结和营销词。每段至少保留一个具体对象或动作，少用“首先/其次/此外”，结尾不要升华。语气按公众号知识文处理，不要写成朋友聊天。

## 禁止滥用的人味

这些词可以用，但不能拿来假装真人：

- “说白了”“老实说”“讲道理”“怎么说呢”
- “真的”“挺”“有点”“其实”
- “对吧”“你懂的”“回见”

使用条件：

- 原文语体允许口语。
- 加入后能推动判断或节奏。
- 不连续堆叠，不替代事实和观点。

## 场景化策略

把抽象句改具体时，优先使用原文已有信息。

坏例子：

> 这款工具能显著提升工作效率。

处理方向：

> 原来要手动整理十几分钟的内容，现在基本变成检查和确认。

如果原文没有足够事实，不要编“我上次用的时候”。可以改成：

> 它真正省下来的，是反复复制、核对、改格式的那一段时间。

## 结构策略

- 原文是“总分总”模板时，可以改成“先给判断 -> 展开原因 -> 用一个具体场景收住”。
- 原文是并列列表时，检查是否真的并列；弱项合并，强项前置。
- 原文每段都在解释概念时，插入问题、反例或具体画面。
- 原文结尾强行升华时，改成一个克制判断或留一个具体问题。

## 公众号开头策略

先判断文章属于哪类开头，再选切法。

### 生活/消费类

先写普通人的小动作，再落到趋势。

- 从杯套、桌面摆件、县城旅行、排队、下单、加班这些具体细节切入。
- 不要一上来写“情绪经济崛起”“消费趋势变化”“生活方式升级”。
- 结尾可以收成一个克制判断，不要强行升华。

### 技术/AI 科普类

先写误解或具体任务，再解释概念。

- 用“买咖啡”“整理表格”“订票”“查资料”这类任务承载概念差异。
- 保留技术清楚感，不要把专业内容全改成聊天。
- 避免“随着技术发展”“正在深刻改变”“开启新时代”。

### 商业/产品分析类

先找产品行为背后的核心张力。

- 小红书这类产品可以写“内容信任和商业转化”的张力。
- 瑞幸这类品牌可以写“规模增长和利润压力”的张力。
- B 站这类社区可以写“社区氛围和商业效率”的张力。
- 不要急着堆“用户需求、盈利模式、商业闭环、运营策略”等维度。

### 观点评论类

先抓读者真实不适感，再给判断。

- AI 焦虑可以写“刚学会一个工具，下一个工具又冒出来”的追赶感。
- 内卷、松弛感、情绪消费这类话题，要先写具体处境，再做判断。
- 需要锋利，但不要靠“不是...而是...”“问题是：”或连续反问制造锋利。

### 动漫剧情解析/短视频口播类

先把事件说清楚，再加情绪和评论。

- 优先写人物动作、人物状态和事件结果，例如“雷道子被王林斩杀”“王平知道自己早已是死人”。
- 可以用短句制造节奏，但短句必须通顺，不能为了“狠”“燃”“刀”把语义压坏。
- 不要把“最狠的是”“不是...而是...”当默认结构。可以改成“这一集的重要看点”“这一集的剧情走向”“这一段真正落在王平身上”等更自然的入口。
- 原作/动画事实要优先于文风。遇到“早死”“散了”“走完最后一步”这类压缩表达，要检查读者是否能准确理解。
- 粉丝向称呼可以保留，例如“各位道友”“小编觉得”，但不要每段都插互动话术。

## 句子逻辑门禁

每句话都要先过逻辑关，再谈去 AI 味。短爆文、观点文、商业分析、技术解释都会出现逻辑不顺；短爆文只是高风险场景，因为它更容易为了锋利感牺牲语义。

### 比较句必须同层级

比较两边必须是同类对象，或者至少能被读者自然放在同一层讨论。

- 错：不是悟道，是护人。
- 对：这段不只讲王林悟道，也讲他怎么处理王平。
- 对：王林这一低头，比赢雷道子更难受。

“悟道”是修行结果，“低头护人”是人物动作，两者直接用“最狠的不是 A，是 B”连接，读者会先卡在逻辑上。

### 强度词必须有对象

“最狠”“封神”“破防”“极致”这类词不能空降。要么前文已经给出比较范围，要么后文有剧情动作支撑。

- 错：最狠的是他第一次低头护人。
- 对：这一集的重要看点，是王林第一次不像只会逆天的修士。
- 对：王林这一低头，比赢雷道子更难受。

### 反转模板不能默认使用

“不是...而是...”只有在 A 和 B 是同类对象，且 B 明确推翻 A 时才保留。否则改成递进、补充或具体判断。

- 错：最狠的不是悟道，是他第一次低头护人。
- 对：二次化凡落下后，王林变的不是境界，是看王平时的那口气。
- 对：这一段真正扎人的地方，是王林终于不只像个逆天修士，也像个父亲。

如果句子删掉“最”“真正”“不是...是...”之后就不成立，说明它靠句式在硬撑。

### 情绪判断要能被动作证明

先写事实、动作或证据，再写情绪判断。写剧情时用剧情动作支撑情绪，写商业分析时用产品行为或数据支撑判断，写技术文时用机制或任务链支撑结论。不要把“狠”“刀”“破防”“关键”“重要”“领先”“革命性”写在前面，后面却没有证据跟上。

- 可支撑的动作：王林收魂、青宜跟着离开、王平知道自己早已是死人、王平主动消散。
- 弱支撑的抽象：悟道、低头、护人、狠、封神。

### 逐句自检

改完后逐句问：

- 这句话有没有比较对象？两边能不能比较？
- 比较两边是不是同一层级？
- 情绪词或强度词有没有事实、剧情动作、产品行为、数据或机制证据？
- 删掉“最、真的、不是...是...”后，句子还成立吗？
- 读者会先被内容打到，还是先被句式绊住？

## 标点节奏

依据 GB/T 15834-2011《标点符号用法》，标点是书面语中表示停顿、语气、结构和成分性质的系统。去 AI 味时不要机械“少用某个符号”，要判断它是不是在替作者搭提纲。

### 停顿层级

常用点号的停顿大致为：句号、问号、叹号最长，冒号次之，分号再短，逗号更短，顿号最短。

使用原则：

- 句号用于意思收住。不要把连续的轻停顿都切成句号，否则会变成一串硬短句。
- 逗号用于句内自然换气。公众号开头可以多用逗号承接观察、补充和转弯，让语气流动起来。
- 顿号只处理并列词语，不要把一串抽象词堆成“效率、创新、赋能、升级”。
- 分号只在复句内部层级真的复杂，尤其分句内部已有逗号时使用。普通文章开头尽量少用。
- 冒号用于真正提示下文、总结上文、引出直接引语或条目字段。不要用来包装“问题是”“结论是”“原因是”。
- 问号用于真实问题或读者会自然追问的位置。不要连续设问，也不要把每段都写成提问式标题。

### AI 常见符号病

- 冒号提纲化：`真正的问题是：...`、`结论是：...`、`主要原因有三：...`
- 句号碎片化：每个短意群都独立成句，读起来像逐条输出。
- 破折号解释癖：一句话刚说完，又用破折号补一个“高级解释”。
- 括号补丁化：把没有融进正文的信息塞进括号，像临时补说明。
- 引号强调癖：普通概念频繁加引号，制造假重点。
- 分号公文腔：用分号串联并列分项，让公众号开头变成报告摘要。

### 修改办法

先改句子，再改标点：

1. 把冒号后的内容改写进自然句。
2. 把过密句号合并成有呼吸的长句。
3. 把破折号和括号里的解释融回正文，融不进去就删除。
4. 把分号列举改成自然段，或只保留最关键的两三个点。
5. 每段读一遍，听停顿是否像人在说话。顿得太满，说明句号太多；气憋太久，说明逗号太多。

对比：

> 智慧家居要进入普通家庭，关键不是未来感，而是三个问题：能不能连上、老人会不会用、坏了谁来修。

更自然：

> 智慧家居要进入普通家庭，光有未来感不够。东西能不能连上，老人会不会用，坏了以后找谁修，这些小事更能决定它能走多远。

再克制一点：

> 智慧家居要进入普通家庭，不能只靠未来感。设备连不上，老人不会用，坏了没人修，任何一个问题都足够让人退回普通家电。

## 语气强度

| 档位 | 用法 |
|------|------|
| 克制 | 商务、技术、正式文章；只删套话和重复 |
| 自然 | 默认；调整段落、句式、连接词和具体性 |
| 犀利 | 评论、社媒、观点文；允许更强判断和短句落点 |

## 质量自检

改完后检查：

- 是否保留了原文事实和核心观点？
- 是否删除了没有信息量的句子？
- 是否还有连续三句结构相同？
- 是否有冒号式结论、破折号解释、括号补丁或过密句号？
- 是否每个句号都对应一次真正收束，而不是轻微停顿？
- 是否为了避开某个 AI 句式，写出了不通顺、不符合事实或不符合中文习惯的句子？
- 比较句是否同层级，强度词是否有事实、动作或机制支撑？
- 是否有至少一处具体对象、动作或场景？
- 是否没有虚构个人经历、数据和来源？
- 是否存在过度拔高、否定式煽情、虚假范围或高频强度词？
- 如果用户提供了样本，是否已经提取并应用了风格锚点，而不是套默认模板？
- 具体数据、引语、文献、独特事实是否可追溯；不可追溯时是否降级或标注待核验？
- 大声读一遍，是否像一个人在表达，而不是模板在运行？
