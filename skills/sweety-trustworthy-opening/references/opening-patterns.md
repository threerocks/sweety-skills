# 公众号可信开头模板

这些模板来自公众号开头方法论资料，但在本 skill 中必须经过可信写作门禁：模板只提供进入正文的路径，不允许夸大、编造、悬空或预支正文。

## 8 种开头模板

| 模板 | 用法 | 可信化要求 |
|------|------|------------|
| 问句式 | 用设问、反问或疑问句开头，引出读者真实困惑 | 问题必须具体，能在正文中回答；避免“你有没有发现一个可怕真相” |
| 对话式 | 把文章里最有信息量的一句话提前 | 对话必须真实或来自正文场景；不要伪造聊天记录 |
| 互动式 | 直接用“你”与目标读者建立关系 | 只指向明确人群和场景；不要用“所有人都”绑架读者 |
| 自我表达式 | 用自嘲、经历或独特处境建立作者可信度 | 经历必须真实可讲；不要卖惨或把普通经历包装成极端遭遇 |
| 题文呼应式 | 第一时间解释标题中的关键词、悬念或判断 | 适合标题有悬念时使用；开头要解题，不要二次悬空 |
| 名人名言式 | 引用、改写俗语、诗句或公共表达 | 标明是戏仿或改写；不要伪造名人真实说过 |
| 热点式 | 用真实热点人物、事件或作品切入 | 热点必须和正文有直接信息增量；不能硬蹭流量 |
| 投稿式 | 用读者来信、留言或提问引出话题 | 只有真实投稿才写投稿；无真实来源时改成“一个常见问题” |

## 3 种增强方法

| 方法 | 用法 | 风险 |
|------|------|------|
| 痛点描述法 | 第一段直接指出目标读者的具体问题 | 痛点过宽会显得像营销课；要绑定具体人群和场景 |
| 故事描写法 | 用小场景、小动作、小冲突开场 | 故事不能脱离正文；不要为了画面感虚构关键事实 |
| 反常识法 | 第一二句话提出和常识不同的判断 | 需要正文证据支撑；不要把“反常识”写成空洞口号 |

## 选择规则

1. 正文有强故事：优先对话式、故事描写法、自我表达式。
2. 正文是观点判断：优先问句式、反常识法、题文呼应式。
3. 正文是方法清单：优先互动式、痛点描述法、题文呼应式。
4. 正文有真实读者材料：可以用投稿式，但要避免泄露隐私。
5. 正文与热点强相关：可以用热点式，但第一段内要交代和正文的关系。

## 三句话检查

写完开头后逐条检查：

- 如果我是目标读者，第一句会让我知道这和我有关吗？
- 它是否承接了标题引发的情绪、疑问或期待？
- 它是否简洁有力，三句话内进入正题？
- 它是否触动了一个具体痛点、冲突或共鸣点？
- 它是否承诺了正文真实能交付的内容？
- 它是否避免了虚假投稿、硬蹭热点和悬空好奇？

## 输出建议

生成候选时，每个模板最多保留 1-2 个候选。最终给用户时优先保留角度差异，而不是把同一个意思换词重复。
