# 标题范式

这些范式吸收自公众号开头写作资料，但只用于标题生成。标题的任务是让读者在 3 秒内看懂主题、情绪方向和点击理由，不负责写标题后的前三句话。

## 可迁移范式

| 范式 | 标题用法 | 风险 |
|------|----------|------|
| 问句式 | 把读者真实困惑压成一个具体问题 | 问题不能太大、太虚或正文无法回答 |
| 互动式 | 用“你是否/你有没有”指向明确人群和场景 | 不要绑架所有读者，不要制造羞辱感 |
| 痛点式 | 直接写出目标读者的具体卡点 | 痛点必须具体，不能像营销课口号 |
| 故事式 | 把正文最有画面的事件压成标题 | 不要为故事感隐瞒主体或结论 |
| 题文呼应式 | 标题给出关键词，正文开头负责解题 | 标题不能只靠悬念成立 |
| 解构式 | 改写俗语、名言或常见表达制造熟悉感 | 不要伪造名人真实说法 |
| 热点式 | 借真实热点切入正文确有增量的话题 | 不能硬蹭热点或名人 |
| 投稿/提问式 | 把真实读者问题压成标题 | 无真实来源时不要写“粉丝投稿” |
| 反常识式 | 写出正文能证明的反直觉判断 | 不要把“反常识”写成空洞口号 |

## 选择规则

1. 正文是方法清单：优先痛点式、问句式、结果式、清单式。
2. 正文是观点判断：优先反常识式、对比式、题文呼应式。
3. 正文是故事或案例：优先故事式、对话感标题、情绪 grounded 标题。
4. 正文有真实读者材料：可以用投稿/提问式，但要避免泄露隐私。
5. 正文与热点强相关：可以用热点式，但标题中必须落回正文对象和观点。

## 3 秒检查

- 读者能不能立刻知道写谁、写什么？
- 读者能不能知道为什么和自己有关？
- 标题有没有给出正文真实能兑现的价值？
- 标题是否只靠悬念、情绪或大词成立？
- 标题和正文开头是否能自然衔接？
